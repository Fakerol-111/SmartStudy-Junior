# 统一 Prompt 配置指南

## 📁 文件位置

所有 AI 教学 Prompt 现已统一存放在 **[config.json](./config.json)**（已大幅扩展）

前后端会自动从这个文件加载配置，**无需分别维护两份**。

## 🎯 最新改进（v2.0）— 问题诊断与解决

### 问题诊断
你说得对，之前的 Prompt **确实太简洁，不够全面**，无法很好回复学生的问题。

**问题原因：**
- ❌ 核心原则只有 5 条，且描述简短
- ❌ 学科提示只有一句话，没有具体教法
- ❌ 任务指令只有一句话，缺少分步指导
- ❌ 没有考虑学生心态（焦虑、自信心不足等）
- ❌ 没有错误处理规则
- ❌ 工具使用指导不详

### 解决方案（v2.0）

#### 1️⃣ **核心教学原则** — 从 5 条 → 7 条（详细版）
```
原来：「苏格拉底式引导：不要直接给答案。先问学生...」
现在：「苏格拉底式引导：不直接给答案，而是通过问题引导学生自己思考。例如：学生问"怎么算这个二次方程"，
      你先问"你看这个方程有什么特点？是否可以因式分解或用公式？"，让学生主动思考。」
```

#### 2️⃣ **学科教法** — 从一句话 → 完整教学方案
每个学科现在包括：
- `description` — 学科定位
- `teaching_approach` — 具体教法（分几步走）
- `common_pitfalls` — 学生常见错误（AI 提前防范）

**示例（数学）：**
```
教法：1) 从具体例子开始，让学生先感受规律
     2) 然后推导通用公式或方法
     3) 最后让学生用这个方法解决变式题
     
常见错误：
- 学生容易混淆正负号 → 讲解时强调符号规则，多给例子
- 方程求解时容易出错 → 让他逐步写出中间步骤
```

#### 3️⃣ **六大任务类型** — 从一句话 → 6-7 步详细规程
每个任务（teach、verify、concept、review、correct、feedback）都有：

**teach（讲解）的完整流程：**
```
第一步 — 理解题意：确保学生知道题问什么、给了什么
第二步 — 激发思考：问"你看这个题有什么特点？"
第三步 — 给出提示而非答案：方向提示或类似例题
第四步 — 完整讲解：如果还是卡住，讲示范解法
第五步 — 巩固：给出1-2道变式题
第六步 — 反馈和鼓励：根据尝试给出反馈
```

#### 4️⃣ **学生心理管理** — 新增 5 种常见情况
```
low_confidence：学生说"我学不会"时
  → 先重建信心，再选简单题让他做对，建立成就感

repeated_questions：学生反复问同一题
  → 不要烦，试试用完全不同的方法讲解

overthinking：学生陷入细节钻牛角尖
  → 帮他"跳出来"，先搞清楚主要思路

rush_or_lazy：学生想偷懒要答案
  → 温和但坚定地拒绝，鼓励他一起思考

curiosity_and_interest：学生问超纲问题
  → 鼓励！即使超纲也可简单解释，满足好奇心
```

#### 5️⃣ **错误处理规则** — 新增 3 种情况
- **ambiguous_question**：题目含糊不清 → 问清楚，不要猜测
- **out_of_scope**：问题超出范围 → 坦诚说，可简单解释
- **no_response**：学生沉默 → 主动问，保持对话活跃

#### 6️⃣ **工具使用指南** — 新增详细场景
```
calculator：什么时候用？
  → 数值计算太复杂或容易出错（开方、大数乘除等）

webSearch：什么时候用？
  → 需要查询最新数据、历史背景、著名人物等

plotter：什么时候用？
  → 分析函数性质、绘制图象、物理运动图象
  
原则：工具是辅助，不是主要讲解。不要让工具替代教学过程。
```

## 🔧 如何修改 Prompt

### 方法 1：直接编辑 JSON（推荐）

1. 打开 `config.json`
2. 修改相应部分：
   - `base.core_principles` — 修改教学原则
   - `base.student_psychology` — 修改心态管理
   - `subjects.{科目}` — 修改某科的教法
   - `intents.{任务}` — 修改某个任务的步骤
3. **保存即可**，下次调用自动生效

### 常见修改示例

**修改数学的教学方法：**
```json
"subjects": {
  "math": {
    "teaching_approach": "我的新教法..."
  }
}
```

**添加新的学生心理情况：**
```json
"base": {
  "student_psychology": {
    "your_case": "学生出现某种情况时..."
  }
}
```

**调整讲解任务的步骤：**
```json
"intents": {
  "teach": {
    "approach": [
      "修改或新增步骤..."
    ]
  }
}
```

## 🔄 前后端加载方式

修改 `config.json` 后，各端自动加载：

- **前端**：`mobile/src/core/systemPrompt.ts` 和 `mobile/src/core/teacher.ts`
  - 自动读取 config.json 中的新结构
  - 支持 objects 和 arrays，自动解析

- **后端**：`backend/app/core/system_prompt.py`
  - 运行时动态读取 config.json
  - 支持 JSON 到 Python dict 的自动转换

修改后建议验证：

1. **格式检查**：JSON 合法性检查
2. **前端验证**：启动应用，与 AI 互动，看教学方式是否改变

## 📌 重要提示

- ✅ JSON 必须合法（可用在线验证器）
- ✅ 文件编码为 **UTF-8**
- ✅ 修改后无需重启编译，下次调用自动应用
- ✅ 过长的 prompt 会增加 API 调用成本，保持简洁
- ✅ `__TOPIC_END__` 规则复杂，修改前理解含义

## 🎓 设计哲学

这个 v2.0 的 Prompt 配置遵循：

1. **具体胜于抽象** — 不说\"要鼓励\"，而是说\"当学生说不会时，先...然后...\"
2. **分步胜于概括** — 不说\"引导式教学\"，而是分 6-7 步详解
3. **场景化** — 针对不同心态、不同任务、不同学科的具体方案
4. **可维护性** — 一个文件，前后端共用，改一处全部生效
